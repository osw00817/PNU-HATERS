## 🔍 참여 방법
1. 이 저장소를 `fork` 한다.
2. 생성된 저장소에 `Github ID`로 폴더를 생성한다.
3. 알고리즘 풀이 후 원본 저장소에 `PR`을 보낸다.

## 📁 Repository 폴더 구조
```
{Github ID}/{week n}/{코드.c}
```
- 💡 예시: `osw00817/week01/BOJ_1374_상근이사랑해.c`

## ⚠️ commit 및 PR 규칙
```
{태그}: {클래스 이름(플랫폼_문제 번호_문제 제목)} {커밋 메시지}
```
- 플랫폼에 따라 없는 내용은 생략 가능
- 💡 예시: `✨feat: PGS_1234_상근이사랑해 알고리즘 구현`

## ✅ 참고해주세요!
### 1. Pull Request하는 법
1) 새로운 branch를 만든다.  
2) 새 branch에 코드를 push한다.  
3) push 완료 후 `contribute`를 누르고 `open pull request`를 선택한다.
4) 원본 저장소를 확인한 후 `create pull request`를 클릭한다.
5) **매일 밤까지 merge**하면 된다. (merge 후 branch는 삭제 가능)
6) 토요일~일요일 스터디시간 전까지 Commit History로 코드리뷰 진행

### 2. Commit History로 리뷰하는 방법
- 다른 사람의 커밋에 댓글을 다는 방식 ([예시](https://github.com/ohgyun/using-github-for-code-reviews/commit/8a85b15805237214aea83a1131f0548b3b69a2d8))

### 3. 코드리뷰 규칙
1) Commit History를 활용한 코드리뷰 진행
2) 코드 흐름을 이해한 후
   - 의견 제시
   - 잘했다고 생각하는 부분
   - 개선 가능성 있는 부분
   - 궁금한 점
   - 대체 풀이 방식 제시
3) 스터디시간에 코드리뷰 내용을 논의
